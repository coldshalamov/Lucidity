# Lucidity Beta Feedback Guide

Thank you for participating in the Lucidity Beta! Your feedback is crucial for making the app better.

## How to Provide Feedback

We primarily use GitHub Issues for tracking bugs and feature requests.

### üêõ Reporting Bugs

Please open an issue in the [GitHub Repository](https://github.com/lucidity-app/lucidity/issues) with the following information:

1.  **Issue Description**: What happened?
2.  **Steps to Reproduce**: Detailed steps so we can see it too.
3.  **Expected Behavior**: What should have happened?
4.  **Actual Behavior**: What actually happened?
5.  **Environment**:
    *   Mobile Device (e.g., Pixel 7, iPhone 14)
    *   OS Version (e.g., Android 14, iOS 17)
    *   Lucidity App Version (check Settings)
    *   Desktop OS (e.g., Windows 11, macOS Sonoma)

### üí° Feature Requests

Have an idea? Let us know!

1.  **Check existing issues**: See if someone else already suggested it.
2.  **Open a new issue**: Select "Feature Request" template.
3.  **Describe the value**: Why is this feature important to you?

### üìß Email

For private feedback or security issues, please email **beta@lucidity.app**.

## Beta Known Issues

*   **Network Switching**: Switching from Wi-Fi to data may cause a delay in reconnection.
*   **Keyboard**: Some specialized terminal key combinations (e.g., Ctrl+Shift+Left) might not be mappable yet.
*   **Performance**: Rendering large amounts of text quickly (e.g., `cat hugefile`) might be choppy on older devices.
